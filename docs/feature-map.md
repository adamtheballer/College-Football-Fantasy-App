## Publication Readiness Rules

- Every league-affecting mutation must emit an `audit_events` row.
- Every write route must require auth and the correct league/team permission.
- Race-prone actions must be transactional and covered by conflict tests.
- Public scoring supports only `QB`, `RB`, `WR`, `TE`, `FLEX`, `SUPERFLEX`, and `K` unless a future product decision explicitly adds more positions.
- Demo/preview UI must never be shown as real league state.
