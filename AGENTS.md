# Project Instructions

## Requirements and Stories
- Any new epic, feature, or story written in `requirements/` should also be published as a GitHub issue or GitHub epic/story set.
- The markdown file in `requirements/` is the repo source of truth for scope and implementation detail.
- The matching GitHub issue should reference the corresponding file in `requirements/`.
- If a requirements document contains multiple implementation stories, those stories should be split into separate GitHub issues for planning and tracking.

## Frontend Direction
- `web/` React is the canonical frontend for this project.
- `ui/` Streamlit is not the primary product surface and should not be used for new feature planning unless explicitly requested.
